# 요구사항 문서

## 소개

FLEX 중국어 시험 대비 AI 학습 에이전트는 공식 교재 부족 문제를 해결하기 위해 LLM 기반으로 FLEX 스타일 독해 문제를 자동 생성하고, 사용자의 풀이 데이터를 분석하여 예상 점수 및 맞춤형 학습 방향을 제공하는 CLI 기반 Python 애플리케이션이다. 본 에이전트는 FLEX 읽기(독해) 영역(600점 만점)에 집중하며, 학습 → 채점 → 분석 → 추천의 반복 루프를 통해 사용자가 목표 점수를 달성할 수 있도록 지원한다.

## 용어 정의

- **Agent**: FLEX 학습 전체 흐름을 조율하는 AI 에이전트 시스템
- **Question_Generator**: LLM을 활용하여 FLEX 스타일 문제를 생성하는 컴포넌트
- **Grader**: 사용자 답안을 채점하고 해설을 제공하는 컴포넌트
- **Analyzer**: 오답 데이터를 분석하여 취약 영역을 도출하는 컴포넌트
- **Recommender**: 취약 영역 기반으로 맞춤 문제를 추천하는 컴포넌트
- **Score_Predictor**: 사용자 성과 데이터를 기반으로 예상 FLEX 점수를 계산하는 컴포넌트
- **Report_Generator**: 학습 리포트를 생성하는 컴포넌트
- **Session**: 사용자가 에이전트와 상호작용하는 단일 학습 세션
- **Session_Store**: 세션 데이터를 JSON 파일로 로컬에 저장하고 불러오는 컴포넌트
- **문제 유형**: 독해(Reading) — FLEX 읽기 영역 전용
- **독해 세부 유형**:
  - 주제 찾기(Topic): 글의 주제, 목적, 제목을 묻는 유형 (上文主要谈的什么? 등)
  - 세부 내용 파악(Detail): 다양한 의문사로 세부 정보를 묻는 유형
  - 진위 판단(True/False): 지문 내용과 일치/불일치 보기를 고르는 유형
  - 편지글(Letter): 비즈니스 편지 및 개인 편지 지문 유형
  - 광고문(Advertisement): 구인·제품·부동산 광고 지문 유형
  - 학술 지문(Academic): 학술적 내용의 세부 사항 및 진위 판단 유형
- **난이도**: 하(Easy), 중(Medium), 상(Hard) 세 단계로 구분하며, 예상 독해 점수 범위에 따라 매핑됨
  - Easy: 0 ~ 350점
  - Medium: 351 ~ 450점
  - Hard: 451 ~ 600점
- **정답률**: 특정 기간 또는 세션 내 전체 문제 수 대비 정답 수의 비율 (0.0 ~ 1.0)
- **예상 점수**: 정답률 및 독해 세부 유형별 가중치를 적용하여 계산한 FLEX 읽기 영역 환산 점수 (0 ~ 600점)
- **목표 점수**: 사용자가 설정한 달성 목표 FLEX 점수

## 요구사항

### 요구사항 1: 학습 세션 초기화

**사용자 스토리:** 학습자로서, 나의 현재 수준 또는 목표 점수를 입력하여 학습 세션을 시작하고 싶다. 그래야 나의 수준에 맞는 문제를 받을 수 있기 때문이다.

#### 인수 기준

1. THE Agent SHALL CLI를 통해 사용자로부터 목표 점수(0 ~ 600 사이의 정수)를 입력받아 Session을 초기화한다.
2. THE Agent SHALL CLI를 통해 사용자로부터 현재 예상 독해 점수(0 ~ 600 사이의 정수)를 입력받아 초기 난이도를 자동 결정한다.
   - 현재 예상 점수 0 ~ 350 → Easy
   - 현재 예상 점수 351 ~ 450 → Medium
   - 현재 예상 점수 451 ~ 600 → Hard
3. IF 사용자가 목표 점수 또는 현재 예상 점수로 0 ~ 600 범위를 벗어난 값을 입력하면, THEN THE Agent SHALL 유효하지 않은 입력임을 알리는 오류 메시지를 출력하고 재입력을 요청한다.
4. IF 사용자가 유효하지 않은 난이도 값을 입력하면, THEN THE Agent SHALL 허용된 난이도 목록을 안내하고 재입력을 요청한다.
5. THE Session_Store SHALL 초기화된 Session 데이터를 로컬 JSON 파일에 저장한다.

---

### 요구사항 2: 문제 생성

**사용자 스토리:** 학습자로서, FLEX 시험 스타일의 4지선다 문제를 자동으로 받고 싶다. 그래야 실전과 유사한 환경에서 연습할 수 있기 때문이다.

#### 인수 기준

1. WHEN 문제 생성 요청이 발생하면, THE Question_Generator SHALL LLM을 호출하여 지정된 독해 세부 유형과 난이도에 맞는 4지선다 문제 1개를 생성한다.
2. THE Question_Generator SHALL 생성된 문제에 문제 본문, 4개의 선택지(①②③④), 정답 번호, 해설을 포함한다.
3. THE Question_Generator SHALL 생성된 문제가 FLEX 읽기 영역 형식(중국어 지문 포함)을 따르도록 LLM 프롬프트를 구성한다.
   - 세부 유형(주제 찾기, 세부 내용 파악, 진위 판단, 편지글, 광고문, 학술 지문) 중 하나를 지정하여 해당 유형에 맞는 지문과 질문을 생성한다.
   - 편지글 유형은 비즈니스 편지 또는 개인 편지 형식을 따른다.
   - 광고문 유형은 구인·제품·부동산 광고 형식 중 하나를 따른다.
   - 학술 지문 유형은 세부 사항 파악 또는 진위 판단 질문을 포함한다.
4. IF LLM 호출이 실패하면, THEN THE Question_Generator SHALL 오류 메시지를 출력하고 재시도 여부를 사용자에게 묻는다.
5. THE Question_Generator SHALL 생성된 문제를 Session_Store에 저장한다.

---

### 요구사항 3: 채점

**사용자 스토리:** 학습자로서, 문제를 풀고 나서 즉시 정답 여부와 해설을 확인하고 싶다. 그래야 틀린 이유를 바로 이해할 수 있기 때문이다.

#### 인수 기준

1. WHEN 사용자가 선택지 번호(1 ~ 4)를 입력하면, THE Grader SHALL 해당 답안과 정답을 비교하여 정오 여부를 판정한다.
2. WHEN 채점이 완료되면, THE Grader SHALL 정답 번호와 해설 텍스트를 CLI에 출력한다.
3. IF 사용자가 1 ~ 4 범위를 벗어난 값을 입력하면, THEN THE Grader SHALL 유효하지 않은 입력임을 알리고 재입력을 요청한다.
4. THE Grader SHALL 채점 결과(문제 ID, 사용자 답안, 정오 여부, 문제 유형, 난이도)를 Session_Store에 저장한다.

---

### 요구사항 4: 오답 분석

**사용자 스토리:** 학습자로서, 내가 자주 틀리는 문제 유형과 취약 영역을 파악하고 싶다. 그래야 어디에 집중해서 공부해야 할지 알 수 있기 때문이다.

#### 인수 기준

1. WHEN 사용자가 오답 분석을 요청하면, THE Analyzer SHALL Session_Store에 저장된 채점 결과를 문제 유형별로 집계하여 유형별 정답률을 계산한다.
2. THE Analyzer SHALL 독해 유형의 경우 세부 유형(주제 찾기, 세부 내용 파악, 진위 판단, 편지글, 광고문, 학술 지문)별 정답률도 함께 집계한다.
3. THE Analyzer SHALL 정답률이 0.6 미만인 문제 유형(및 독해 세부 유형)을 취약 영역으로 분류한다.
4. THE Analyzer SHALL 취약 영역 목록과 각 유형별 정답률을 CLI에 출력한다.
5. IF 분석에 사용할 채점 데이터가 5개 미만이면, THEN THE Analyzer SHALL 분석에 충분한 데이터가 없음을 안내하고 더 많은 문제 풀이를 권장한다.

---

### 요구사항 5: 맞춤 문제 추천

**사용자 스토리:** 학습자로서, 내 취약 영역 중심의 문제를 추천받고 싶다. 그래야 효율적으로 약점을 보완할 수 있기 때문이다.

#### 인수 기준

1. WHEN 맞춤 문제 추천이 요청되면, THE Recommender SHALL Analyzer가 도출한 취약 영역 중 정답률이 가장 낮은 문제 유형을 우선 선택하여 Question_Generator에 문제 생성을 요청한다.
2. WHERE 취약 영역이 2개 이상인 경우, THE Recommender SHALL 취약 영역 간 문제 유형을 순환하여 추천한다.
3. IF 취약 영역이 존재하지 않으면, THEN THE Recommender SHALL 현재 난이도보다 한 단계 높은 난이도의 문제를 균등하게 추천한다.
4. THE Recommender SHALL 추천 근거(어떤 취약 영역을 보완하기 위한 문제인지)를 CLI에 출력한다.

---

### 요구사항 6: 난이도 자동 조절

**사용자 스토리:** 학습자로서, 내 실력 변화에 따라 문제 난이도가 자동으로 조정되기를 원한다. 그래야 항상 적절한 수준의 도전을 받을 수 있기 때문이다.

#### 인수 기준

1. WHEN 연속 5문제 이상의 채점 결과가 누적되면, THE Agent SHALL 최근 5문제의 정답률을 계산하여 난이도 조정 여부를 판단한다.
2. WHEN 최근 5문제 정답률이 0.8 이상이면, THE Agent SHALL 현재 난이도를 한 단계 상향 조정한다.
3. WHEN 최근 5문제 정답률이 0.4 미만이면, THE Agent SHALL 현재 난이도를 한 단계 하향 조정한다.
4. WHILE 현재 난이도가 Hard이면, THE Agent SHALL 정답률이 0.8 이상이더라도 난이도를 상향 조정하지 않는다.
5. WHILE 현재 난이도가 Easy이면, THE Agent SHALL 정답률이 0.4 미만이더라도 난이도를 하향 조정하지 않는다.
6. WHEN 난이도가 변경되면, THE Agent SHALL 변경된 난이도와 해당 난이도에 대응하는 예상 점수 범위를 사용자에게 CLI로 알리고 Session_Store에 저장한다.
7. THE Agent SHALL 예상 독해 점수가 갱신될 때마다 해당 점수 범위에 맞는 난이도와 현재 설정된 난이도를 비교하여 불일치 시 사용자에게 난이도 재조정을 제안한다.

---

### 요구사항 7: 예상 점수 예측

**사용자 스토리:** 학습자로서, 현재 실력을 기반으로 예상 FLEX 점수를 확인하고 싶다. 그래야 목표 점수 달성 가능성을 객관적으로 파악할 수 있기 때문이다.

#### 인수 기준

1. WHEN 예상 점수 계산이 요청되면, THE Score_Predictor SHALL 전체 정답률에 600을 곱하여 기본 예상 점수를 계산한다.
2. THE Score_Predictor SHALL 독해 세부 유형별 가중치를 적용한 가중 예상 점수를 계산한다 (각 세부 유형의 가중치는 균등 배분을 기본으로 하며 추후 조정 가능).
3. THE Score_Predictor SHALL 기본 예상 점수와 가중 예상 점수를 모두 600점 만점 기준으로 CLI에 출력한다.
4. IF 계산에 사용할 채점 데이터가 10개 미만이면, THEN THE Score_Predictor SHALL 데이터 부족으로 인해 예상 점수의 신뢰도가 낮을 수 있음을 경고 메시지와 함께 출력한다.
5. THE Score_Predictor SHALL 계산된 예상 점수를 Session_Store에 저장한다.

---

### 요구사항 8: 학습 리포트 생성

**사용자 스토리:** 학습자로서, 나의 학습 현황을 종합적으로 정리한 리포트를 받고 싶다. 그래야 강점과 약점을 한눈에 파악하고 다음 학습 방향을 결정할 수 있기 때문이다.

#### 인수 기준

1. WHEN 학습 리포트 생성이 요청되면, THE Report_Generator SHALL Score_Predictor와 Analyzer의 결과를 종합하여 리포트를 생성한다.
2. THE Report_Generator SHALL 리포트에 현재 예상 점수, 목표 점수, 목표 점수 대비 달성률(예상 점수 / 목표 점수 × 100)을 포함한다.
3. THE Report_Generator SHALL 리포트에 문제 유형별 정답률을 포함하여 강점 영역(정답률 0.7 이상)과 약점 영역(정답률 0.6 미만)을 구분하여 표시한다.
4. THE Report_Generator SHALL 리포트에 취약 영역 개선을 위한 구체적인 학습 방향 텍스트를 포함한다.
5. THE Report_Generator SHALL 생성된 리포트를 CLI에 출력하고 Session_Store에 저장한다.
6. IF 리포트 생성에 필요한 채점 데이터가 존재하지 않으면, THEN THE Report_Generator SHALL 학습 데이터가 없음을 안내하고 문제 풀이를 먼저 진행하도록 안내한다.

---

### 요구사항 9: 학습 데이터 저장 및 불러오기

**사용자 스토리:** 학습자로서, 이전 학습 세션의 데이터가 유지되기를 원한다. 그래야 누적된 데이터를 기반으로 더 정확한 분석을 받을 수 있기 때문이다.

#### 인수 기준

1. THE Session_Store SHALL 모든 세션 데이터를 로컬 JSON 파일(`session_data.json`)에 저장한다.
2. WHEN Agent가 시작되면, THE Session_Store SHALL 기존 `session_data.json` 파일이 존재하는 경우 해당 데이터를 불러와 Session을 복원한다.
3. IF `session_data.json` 파일이 손상되거나 파싱에 실패하면, THEN THE Session_Store SHALL 오류를 기록하고 빈 Session으로 초기화한다.
4. THE Session_Store SHALL 데이터 저장 시 기존 파일을 덮어쓰기 전에 임시 파일에 먼저 기록한 후 교체하여 데이터 손실을 방지한다.

---

### 요구사항 10: 반복 학습 루프

**사용자 스토리:** 학습자로서, 학습 → 채점 → 분석 → 추천의 흐름이 자연스럽게 반복되기를 원한다. 그래야 별도의 조작 없이 지속적으로 학습을 이어갈 수 있기 때문이다.

#### 인수 기준

1. WHEN 한 문제의 채점이 완료되면, THE Agent SHALL 사용자에게 다음 행동(다음 문제 풀기, 오답 분석, 리포트 보기, 종료)을 선택하도록 메뉴를 제공한다.
2. WHEN 사용자가 다음 문제 풀기를 선택하면, THE Agent SHALL Recommender를 통해 다음 문제를 생성하고 출력한다.
3. WHILE 사용자가 종료를 선택하지 않는 한, THE Agent SHALL 학습 루프를 유지한다.
4. WHEN 사용자가 종료를 선택하면, THE Agent SHALL 최종 학습 리포트를 출력한 후 세션을 종료한다.
